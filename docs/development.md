# Development notes

## Page structure
Most pages are standalone HTML files. Keep them structured in a way that is easy to read and easy to duplicate for new stories.

## Bilingual content
When creating a new story, create matching pages in both:
- `en/`
- `fr/`

This helps preserve a consistent reader experience across both language versions.

## Assets
Place new images in the `images/` directory and reference them using relative paths that work from the page's folder.

## Shared pieces
If you need to update the site header, language toggle, or repeated page elements, prefer changing the shared assets rather than copying the same markup into many pages.
